# 需求管理平台后端领域入口

## 业务边界

本领域承接需求管理平台后端能力，包括项目、仓库、客户定制线、模块功能点、项目索引知识库、需求、Agent 交接资料、MCP、业务事件和统计。

## 当前状态

- 业务模块 `ruoyi-requirement` 已创建，承载需求平台核心后端能力。
- MVP-lite 已具备项目、仓库、客户线、模块、需求、执行包、MCP 和统计基础接口。
- 项目管理已具备初始化聚合接口，管理员可通过 `/requirement/project/init/**` 一次性新增或编辑项目、前后端仓库和分支配置，并读取模块摘要、索引摘要和初始化检查项。分支配置对需求人员展示中文标签，对索引和开发流程保存真实分支名。
- 当前增强方向是项目接入中心和本地仓库索引闭环：本地 agent 读取开发者机器上的仓库，通过 MCP `publish_repository_index` 上传结构化索引结果；平台只保存相对路径、Git 分支、commit、客户线或基线分支和影响面，不保存个人本机目录。
- 新需求创建时，后端按所选客户线解析统一基线分支，并从目标仓库或每个仓库最新导入批次中推荐页面、接口、数据表、权限和相关文档。

## 后续维护

- 新增接口、数据库表、SQL 口径或权限标识时，同步补充 `docs/ai-harness/contracts/` 或 `docs/db/`。
- 新增长期有效业务决策时，同步补充 `docs/ai-harness/decisions/`。
- 调整 MCP tools 或项目索引字段时，同步更新接口契约和数据库关系说明。
