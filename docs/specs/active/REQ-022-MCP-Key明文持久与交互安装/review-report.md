## 风险说明

- 升级前历史 Key 没有明文，无法恢复；需要用户重新生成 Key。
- 明文持久化本身是明确产品选择，后续必须避免把 `plainKey` 写入操作日志、活动记录、本地存储或列表字段。
- MCP 请求地址只保存 host/port，完整地址由后端按协议、context-path 和 `/requirement/mcp` 生成；部署时不能把前端静态项目名当作 MCP 路径前缀。
