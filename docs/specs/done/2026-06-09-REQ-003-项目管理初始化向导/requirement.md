# 项目管理初始化向导后端需求说明

## 背景

REQ-002 已完成 MCP 索引入口、索引知识库和影响面推荐基础能力，但没有完整实现最初设想的后台项目管理菜单体验。当前管理员仍需要在项目、仓库、客户线、模块等多个入口之间切换，才能完成一个项目从创建到可索引、可提需求的初始化工作。

目标产品形态要求“项目管理”本身就是项目初始化入口：管理员在项目管理菜单内新增或编辑项目时，可以在一个维护弹窗内维护项目名称、前端仓库、后端仓库和后续可持续新增的分支列表，并能查看模块功能点初始化状态。平台仍不得保存个人本机绝对路径，只保存团队共享的 Git 远端、真实分支名、commit、相对路径和结构化模块信息。分支对需求人员展示中文标签，对索引和开发流程使用真实分支名。

## 目标

- 提供项目初始化聚合契约，支持一次性读取项目基础信息、仓库、分支列表、模块初始化状态和索引概览。
- 提供项目初始化保存能力，支持在同一事务内新增或更新项目、前后端仓库和分支配置。
- 提供模块初始化状态口径，让前端能够提示管理员哪些仓库已索引、哪些模块已沉淀、哪些项目仍未完成初始化。
- 保持现有项目、仓库、客户线和模块 CRUD 可用，不破坏 REQ-002 的 MCP 索引和影响面推荐契约。
- 明确后端不接收、不保存个人本机目录；本机仓库路径仅属于本地索引器执行时的临时输入。

## 范围

本次包含：

- 新增项目初始化 DTO，用于承载项目基础信息、仓库列表、分支列表、模块初始化摘要和索引状态。
- 新增聚合查询接口，供项目管理菜单打开新增/编辑向导或项目详情时一次性加载初始化上下文。
- 新增聚合保存接口，支持保存项目基础信息、前后端仓库、分支中文标签、真实分支名和初始化备注。
- 增加后端校验：同一项目下仓库类型、仓库地址、分支中文标签、真实分支名不能为空且不能包含个人绝对路径。
- 更新接口契约、数据库关系说明和领域文档，说明项目初始化的数据粒度和边界。

本次不包含：

- 平台服务器主动 clone 仓库或扫描文件系统。
- 在后端执行 Git、shell、分支创建或代码提交。
- 浏览器上传本机仓库绝对路径到数据库。
- 完整模块解析算法；模块功能点仍由 MCP 索引沉淀或管理员在模块管理中校正。
- 重新设计需求状态机、Agent 交接资料和统计模块。

## 影响范围

- 接口/API：是，新增项目初始化聚合查询和聚合保存接口。
- 数据库/SQL：原则上不新增核心表；优先复用 `req_project`、`req_repository`、`req_variant`、`req_module`、`req_repository_index_batch`。如实现时发现缺少初始化状态字段，必须先更新计划和 DDL。
- 权限/菜单：是，初始化查询和保存需要复用或组合 `req:project:*`、`req:repo:*`、`req:variant:*` 权限。
- 页面/交互：是，前端 companion 将项目管理菜单改为初始化向导。
- 导出/异步/任务：否。

## 契约与数据口径

- 建议接口：
  - `GET /requirement/project/init/{projectId}`：查询已有项目初始化上下文。
  - `POST /requirement/project/init`：新增项目并保存仓库、分支列表。
  - `PUT /requirement/project/init`：更新项目初始化信息。
- 请求体建议：
  - `project`：项目名称、项目编码、负责人、说明、状态。
  - `repositories`：仓库名称、仓库类型、Git 远端地址、默认分支、状态。前端至少支持一条前端仓库和一条后端仓库，也允许后续扩展多个仓库。
  - `variants`：分支中文标签 `branchLabel`、真实分支名 `baselineBranch`、状态、备注。为兼容既有 `req_variant`，响应仍保留 `variantName`、`variantCode`、`customerName`、`scopeType`、`branchPolicy` 字段；后端可在 `variantCode` 为空时按真实分支名兜底生成稳定编码。
  - `remark`：初始化备注或管理员说明。
- 响应字段建议：
  - `project`：项目实体。
  - `repositories`：项目下仓库列表。
  - `variants`：项目下分支配置列表。
  - `moduleSummary`：模块总数、索引模块数、人工模块数。
  - `indexSummary`：最近索引时间、最近 commit、已索引仓库数、未索引仓库数。
  - `initChecklist`：`projectReady`、`repositoryReady`、`variantReady`、`moduleReady`、`indexReady`。
- 数据粒度：
  - 初始化上下文一次返回一个项目的初始化全貌。
  - 仓库列表一行代表一个团队共享代码仓库。
  - 分支列表一行代表一个可供需求人员选择的项目分支配置；中文标签面向需求人员展示，真实分支名用于索引、影响面推荐和后续开发基准。
  - 模块初始化摘要只表达当前状态，不替代模块明细 CRUD。

## 验收标准

- AC-BE-001：后端提供项目初始化上下文查询接口，能一次性返回项目、仓库、分支配置、模块摘要和索引摘要。
- AC-BE-002：后端提供项目初始化聚合保存接口，新增项目时可同时保存至少前端仓库、后端仓库和一条分支配置。
- AC-BE-003：聚合保存具备事务一致性；项目、仓库或分支配置任一校验失败时不得留下部分初始化数据。
- AC-BE-004：后端拒绝仓库地址、默认分支、真实分支名或备注中的个人本机绝对路径。
- AC-BE-005：初始化接口具备权限控制，新增使用 `req:project:add`、更新使用 `req:project:edit`，仓库和客户基线写入同时校验对应维护权限。
- AC-BE-006：后端 harness 文档同步更新接口契约、数据库关系和领域说明。

## 约束与假设

- REQ-003 依赖 REQ-002 的仓库、客户基线、索引批次和影响面数据结构。
- 本需求已从 REQ-002 提交后的新分支启动；执行代码前仍需确认后端既有配置文件改动是否纳入后续提交。
- 后端第一版优先实现聚合读取和聚合保存，不引入新的流程引擎或任务调度。
- 初始化状态优先从现有表派生，避免新增冗余状态字段。
