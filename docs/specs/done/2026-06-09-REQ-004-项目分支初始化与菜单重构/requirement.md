# 项目分支初始化与菜单重构后端需求说明

## 目标

将后端长期契约从“仓库管理、客户定制线、模块功能点独立 CRUD”调整为“项目仓库、项目分支、索引知识”的内部模型。初始化保存按项目分支记录沉淀情况，允许纯后端服务，并为 MCP 生成稳定识别 key。

## 范围

- 项目初始化保存只要求至少一条有效代码仓库和至少一条项目分支，不再强制同时存在前端仓库和后端仓库。
- 项目分支返回 `mcpKey`，用于 MCP 识别项目和分支。
- `publish_repository_index` 支持使用 `mcpKey + remoteUrl` 解析项目、分支和仓库；兼容旧的 `projectId + repoId` 调用方式。
- 菜单 SQL 不再创建仓库管理、客户定制线和模块功能点菜单，并提供对既有菜单的下线处理。
- 项目初始化接口权限收敛为项目权限，避免下线旧菜单权限后无法初始化。
- 后端契约、数据库关系和领域文档同步把“客户定制线”长期语义调整为“项目分支”。

## 不做范围

- 不物理删除 `req_repository`、`req_variant`、`req_module` 表。
- 不删除已有 Controller 和 Service，旧接口作为内部兼容能力保留，后续如需公开废弃再单独处理。
- 不执行 Git、shell、clone 或本机文件扫描。
- 不保存个人本机绝对路径。

## 验收标准

- AC-BE-001：纯后端项目只维护一个后端仓库时，项目初始化保存通过。
- AC-BE-002：初始化状态按至少一个有效仓库判断仓库就绪，不再要求前端+后端。
- AC-BE-003：分支配置生成并返回稳定 `mcpKey`。
- AC-BE-004：MCP 索引导入可通过 `mcpKey + remoteUrl` 定位项目、分支和仓库。
- AC-BE-005：项目初始化新增和修改只依赖项目新增/修改权限。
- AC-BE-006：菜单 SQL 不再创建仓库管理、客户定制线和模块功能点。
- AC-BE-007：后端 harness、数据库关系和领域文档同步新语义。
