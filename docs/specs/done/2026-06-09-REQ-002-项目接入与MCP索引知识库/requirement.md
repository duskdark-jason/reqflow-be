# 项目接入与 MCP 索引知识库后端需求说明

## 背景

现有需求管理平台 MVP-lite 已提供项目、仓库、客户线、模块功能点、需求和执行包等基础 CRUD，但项目管理与目标产品形态仍有差距：项目、仓库、客户基线和模块知识分散维护，模块功能点依赖管理员手工录入，创建需求时不能根据模块自动推荐影响页面、接口、数据表和权限点。

目标设计要求平台能读取本机前后端仓库做索引，但不能把个人机器上的绝对目录保存到平台。多人开发时，每个人的大目录不同，平台只能保存可共享的仓库身份、相对路径、分支、commit 和结构化索引结果。本需求采用 MCP 作为推荐上传入口：本地 Codex 或索引 Agent 在开发者机器读取仓库，调用平台 MCP tool 推送索引结果；平台沉淀为项目知识库，供后续新需求自动匹配影响面。

## 目标

- 将项目接入数据模型从分散 CRUD 升级为“项目、仓库、客户基线、知识索引”的闭环。
- 提供 `publish_repository_index` MCP tool，用于接收本地索引器上传的仓库索引结果。
- 后端保存仓库相对路径、模块、页面、接口、数据表、权限、文档摘要、分支和 commit，不保存本机绝对路径。
- 创建或编辑需求时，支持根据项目、客户线、模块或功能点查询推荐影响面。
- 将“需求执行包”的业务语义明确为开发阶段的 Agent 交接资料，不作为需求人员必须理解的核心对象。

## 范围

本次包含：

- 新增或调整索引相关数据库表，用于保存模块知识库、影响面条目和索引批次。
- 扩展 MCP `tools/list` 与 `tools/call`，新增 `publish_repository_index`。
- 新增索引导入 Service，校验上传结果不包含本机绝对路径，并按仓库、分支、commit 版本化。
- 新增需求影响面推荐接口，按项目、客户线、模块或功能点返回页面、接口、数据表、权限和相关文档。
- 更新后端接口契约、数据库关系文档和领域文档。

本次不包含：

- 平台服务器主动 clone 仓库或保存 Git 凭证。
- 平台后端直接扫描服务器文件系统。
- 自动创建 Git 分支、提交代码、创建合并请求。
- 在平台内调用大模型生成索引。
- 替换现有需求状态流转和执行包版本追加规则。

## 影响范围

- 接口/API：是，新增 MCP tool 和索引查询、影响面推荐接口。
- 数据库/SQL：是，新增或调整索引批次、模块知识、影响面条目相关表。
- 权限/菜单：是，新增索引发布和索引查询权限，MCP 写入仍需服务端权限保护。
- 页面/交互：是，前端 companion 负责项目接入中心、索引入口和需求影响面推荐展示。
- 导出/异步/任务：否，本次不引入后台任务调度；索引由本地 Agent 读取后推送。

## 契约与数据口径

- MCP 方法：继续使用 `POST /requirement/mcp` 的 JSON-RPC 风格入口。
- MCP tool：`publish_repository_index`。
- REST 接口建议：
  - `GET /requirement/index/batch/list` 查询索引批次。
  - `GET /requirement/index/module/tree` 查询项目模块知识树。
  - `GET /requirement/index/impact/suggest` 按项目、客户线、模块或功能点推荐影响面。
  - `POST /requirement/index/import` 作为页面上传 JSON 的备用入口。
- MCP tool 参数：
  - `projectId`：平台项目 ID。
  - `repoId`：平台仓库 ID。
  - `repoType`：仓库类型，例如 `FRONTEND`、`BACKEND`。
  - `remoteUrl`：Git 远端地址，用于校验仓库身份。
  - `branchName`：索引分支。
  - `commitHash`：索引 commit。
  - `indexVersion`：索引格式版本。
  - `modules`、`pages`、`apis`、`tables`、`permissions`、`documents`：结构化索引内容。
- 禁止字段：上传内容不得包含 macOS、Linux 或 Windows 用户目录形式的个人绝对路径；如发现应拒绝导入并返回明确错误。
- 数据粒度：
  - 索引批次一行代表某仓库某分支某 commit 的一次上传。
  - 模块知识一行代表一个平台可选模块或功能点。
  - 影响面条目一行代表一个页面、接口、数据表、权限或文档资源。

## 验收标准

- AC-BE-001：平台提供 `publish_repository_index` MCP tool，并能接收本地 Agent 上传的仓库索引结果。
- AC-BE-002：索引导入拒绝包含个人本机绝对路径的数据，平台数据库不保存本机目录。
- AC-BE-003：索引结果按仓库、分支、commit 记录批次，并能更新项目模块知识库。
- AC-BE-004：后端提供影响面推荐接口，可按项目和模块返回页面、接口、数据表、权限和相关文档。
- AC-BE-005：索引导入和推荐接口具备权限控制、参数校验和审计事件记录。
- AC-BE-006：后端 harness 文档同步更新数据库关系、接口契约和需求平台领域说明。

## 约束与假设

- 本地索引器由 Codex 或后续脚本在开发者机器执行，平台只接收结构化结果。
- MCP 是推荐上传入口；REST JSON 导入仅作为备用入口，便于调试和人工补录。
- 本需求不要求一次性支持所有技术栈的完美扫描，第一版以现有 RuoYi Vue 前端和 Spring/MyBatis 后端可识别结构为优先。
- 管理员可以在平台校正索引生成的模块和影响面，校正结果应保留人工维护能力。
