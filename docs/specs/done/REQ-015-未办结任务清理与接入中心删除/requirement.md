# 未办结任务清理与接入中心删除需求说明

## 背景

当前后端仓库仍有已合入主干但未完成归档的 active spec，导致“已完成事项”和“仍在执行事项”混在一起。前端项目管理里，“项目接入中心”和“项目维护”职责继续重叠，且分支知识库已经由维护路径和独立知识库页签承接。

## 目标

- 清理已合入主干但仍停留在 `docs/specs/active` 的历史任务。
- 删除前端“项目接入中心”页面链路，项目列表只保留项目维护入口。
- 更新需求平台模块知识库，明确项目维护和分支知识库是后续维护入口。
- 保持后端接口和权限标识稳定，避免引入不必要兼容层。

## 范围

本次包含：

- 归档 `REQ-012-CodexSkillYAML修复`、`REQ-013-初始化指令简化与索引表核对`、`REQ-014-harness命名规则移除日期`。
- 更新后端模块知识库中关于项目接入中心、项目维护和分支知识库的说明。
- 与前端 companion spec 保持同一任务分支和同一验收口径。

本次不包含：

- 修改后端业务实现或返回字段。
- 直接调整本地平台库中的需求流转状态。
- 删除项目初始化上下文接口，因为项目维护和分支知识库仍使用该接口。
- 自动合并、推送或删除远端分支。

## 影响范围

- 接口/API：否，后端接口路径和字段不变。
- 持久化结构/脚本：否。
- 权限/菜单：否，后端权限标识不变；前端删除隐藏页签入口。
- 页面/交互：是，前端删除“项目接入中心”页面链路。
- 导出/异步/任务：否。

## 契约与数据口径

- 接口路径和方法：无新增或删除。
- 请求参数：无变化。
- 响应字段：无变化。
- 数据粒度：一个 spec 目录表示一项需求过程文档；完成后应归档到 `docs/specs/done`。

## 验收标准

- AC-001：已合入主干的 `REQ-012`、`REQ-013`、`REQ-014` 不再停留在 `docs/specs/active`。
- AC-002：后端模块知识库不再把“项目接入中心”记录为独立长期入口，项目初始化状态和分支知识库归入项目维护相关说明。
- AC-003：后端文档检查和 harness 完成态检查通过。
- AC-004：前端 companion 删除“项目接入中心”入口、隐藏路由和页面文件，并通过静态检查与构建验证。

- “项目接入中心”删除是页面链路删除，不等于删除项目初始化上下文接口。
